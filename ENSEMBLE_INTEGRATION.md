# 🤖 Интеграция ансамбля моделей в NutriBuddy Bot

## ✅ Что было сделано

### 1. **Добавлена функция `identify_food_ensemble`**
- Запускает несколько vision-моделей параллельно
- Использует взвешенное голосование для выбора лучшего результата
- Поддерживает кэширование и прогресс-бары
- По умолчанию использует LLaVA + UForm

### 2. **Обновлены основные функции**
- `identify_food()` → теперь использует ансамбль
- `identify_food_cascade()` → теперь использует ансамбль  
- `get_simple_ingredients()` → оптимизирована для скорости
- `identify_dish_from_image()` → обратно совместима

### 3. **Улучшена обработка данных**
- Взвешенное голосование на основе уверенности моделей
- Пост-обработка и исправление ошибок
- Кэширование результатов ансамбля
- Расширенное логирование процесса

## 🎯 Преимущества ансамбля

### **Точность распознавания**
- **LLaVA** (вес 0.6): Лучше понимает контекст и сложные блюда
- **UForm** (вес 0.4): Быстрее и хорошо выделяет ингредиенты
- **Комбинация**: Снижает риск пропустить важные детали

### **Надежность**
- Если одна модель fails, другая может справиться
- Взвешенное голосование выбирает наиболее достоверный результат
- Параллельное выполнение уменьшает общее время

### **Гибкость**
- Можно выбирать разные комбинации моделей
- Поддержка кастомных весов моделей
- Оптимизация для скорости или точности

## 📊 Пример использования

```python
# Базовый ансамбль (LLaVA + UForm)
data, model = await identify_food_ensemble(image_bytes)

# Только LLaVA для максимальной точности
data, model = await identify_food_ensemble(image_bytes, model_indices=[0])

# Только UForm для скорости  
data, model = await identify_food_ensemble(image_bytes, model_indices=[1])

# Все три модели для максимальной надежности
data, model = await identify_food_ensemble(image_bytes, model_indices=[0,1,2])
```

## 🔧 Технические детали

### **Модели и их веса**
```python
VISION_MODELS = [
    {"id": "@cf/llava-hf/llava-1.5-7b-hf", "priority": 1, "timeout": 90, "weight": 0.6},
    {"id": "@cf/unum/uform-gen2-qwen-500m", "priority": 2, "timeout": 60, "weight": 0.4},
    {"id": "@cf/llava-hf/bakllava-1", "priority": 3, "timeout": 90, "weight": 0.3},
]
```

### **Алгоритм голосования**
1. Параллельный запуск всех выбранных моделей
2. Валидация результатов каждой модели
3. Вычисление взвешенной оценки: `confidence * weight`
4. Выбор результата с максимальной оценкой
5. Пост-обработка и кэширование

### **Обработка ошибок**
- Если модель fails → пропускается
- Если все модели failed → fallback к старой системе
- Исключения логируются с полным traceback
- Graceful degradation при проблемах
